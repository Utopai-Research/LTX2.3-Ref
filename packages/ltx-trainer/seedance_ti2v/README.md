# LTX-2.3 reference-to-video (R2V) fine-tune — Seedance

Fine-tune of the **LTX-2.3 (22B) audio-video model** to generate video from a **text
prompt + up to 9 reference images** (characters/objects/scenes — no first frame).
Each reference image is VAE-encoded and concatenated **in-context** with a learned
per-image identity embedding; prompts refer to them as `[Image1]`, `[Image2]`, ...

## Inference

`inference_refs.py` generates one video (with audio) from a prompt + 0-9 reference
images with a trained checkpoint — full-FT or LoRA, auto-detected. With no
`--ref-images` it runs pure text-to-video.

```bash
# 0) environment (from the repo root; Linux + CUDA GPU with >=80 GB VRAM):
uv sync
cd packages/ltx-trainer

# 1) base models (LTX-2.3 dev ~44 GB + Gemma-3 encoder ~24 GB, gated on HF):
bash seedance_ti2v/download_models.sh

# 2) trained checkpoint (v2 full-FT, step 28000, ~38 GB) + its training_config.yaml,
#    both from this Google Drive folder:
#      https://drive.google.com/open?id=1kXmB_WONRASvEEJovxFh1pZhSkISvWRP
#    - model_weights_step_28000.safetensors:
#      https://drive.google.com/open?id=1vdEDBJRSGU3kXYR2RGYaVK4KCWPI554w
#    - training_config.yaml:
#      https://drive.google.com/open?id=1hejQDUgTt-iXJ59A2np2hVuLxHhbUx86
#    keep training_config.yaml NEXT TO the checkpoint — the script auto-reads the
#    reference-conditioning scheme from it.

# 3) generate (defaults = v2 model: 1280x704 @ 24 fps, aspect-bucketed refs):
../.venv/bin/python seedance_ti2v/inference_refs.py \
    --checkpoint model_weights_step_28000.safetensors \
    --base-checkpoint /path/to/LTX-2.3/ltx-2.3-22b-dev.safetensors \
    --text-encoder-path /path/to/gemma-3-12b-it-qat-q4_0-unquantized \
    --ref-images ref0.png ref1.png \
    --prompt "The character in [Image1] talks to the character in [Image2] in a snowy street." \
    --num-frames 121 --output out.mp4
```

Notes:
- Reference order matters: the first `--ref-images` path is `[Image1]` in the prompt, etc.
- `--num-frames` must satisfy `N % 8 == 1` (121 = 5 s; the script snaps down automatically).
- Audio is generated by default; `--skip-audio` disables it. `--seed` for variations.
- Expect several minutes of model loading per invocation, then a few minutes per 5 s clip
  on an H100/H200. For batch generation over a test set (multi-GPU/multi-node, resumable)
  use `run_eval_refs.py`.

## Training pipeline

Training uses precomputed latents. Dataset manifest is a JSONL with per-clip
`video`, `caption`, and `ref_images` (list of image paths).

```bash
cd packages/ltx-trainer

# 1) dataset prep: filter 16:9 clips, train/test split
#    (seedance: prepare_dataset.py; cue_prod export: export_cueprod_manifest.py)
../.venv/bin/python seedance_ti2v/prepare_dataset.py --num-test 50 --seed 42

# 2) precompute: text embeddings + audio latents (v1 root), then the v2 hires root
bash seedance_ti2v/run_preprocess.sh              # Gemma text embeds + audio latents
bash seedance_ti2v/run_preprocess_v2_setup.sh     # v2 root; symlinks resolution-independent parts
bash seedance_ti2v/run_preprocess_v2_video.sh     # 1280x704 video latents (single node)
#   or multi-node: sbatch seedance_ti2v/run_preprocess_v2_video_4node.sbatch
OUT_DIR=<v2_root>/reference_image_latents bash seedance_ti2v/run_preprocess_refs.sh   # aspect-bucketed refs

# 3) train: full FT with FSDP/HSDP over 4 nodes (8xH200 each), Slurm
sbatch seedance_ti2v/run_train_refs_full_v2_4node.sbatch     # config: ltx2.3_reference_full_v2.yaml
```

The cue_prod large-dataset variants (`*_cueprod_*.sbatch`, `ltx2.3_reference_cueprod_v2.yaml`)
are the same recipe on ~13.4k clips.

## Files

| file | what |
|---|---|
| `inference_refs.py` | single-video inference: prompt + reference images -> mp4 (+audio) |
| `run_eval_refs.py` | batch eval harness (sharded, resumable, prompt-embedding cache) |
| `download_models.sh` | fetch LTX-2.3 `dev` checkpoint + Gemma-3 encoder |
| `prepare_dataset.py` / `export_cueprod_manifest.py` | dataset manifests (seedance / cue_prod) |
| `run_preprocess*.sh/.sbatch` | precompute video/ref latents + text/audio (v2 hires pipeline) |
| `ltx2.3_reference_full_v2.yaml` | training config of the released checkpoint |
| `ltx2.3_reference_cueprod_v2.yaml` | same recipe, cue_prod large dataset |
| `run_train_refs_*.sbatch` | multi-node Slurm launchers (FSDP/HSDP) |

## Key training settings (`ltx2.3_reference_full_v2.yaml`)

- `training_strategy.name: reference_images` — in-context ref tokens, learned identity
  embedding, shared ref RoPE time `-1.0`, `max_reference_slots: 9`.
- `reference_dropout_p: 0.1` — 10% of samples drop all refs; trains the ref-less branch
  (enables pure T2V and `--ref-guidance-scale` at inference).
- LR 2e-5 constant (global batch 32 across 4 nodes), bf16, gradient checkpointing,
  checkpoint every 500 steps.

## Caveats

- **Prompt format**: training prompts reference images as `[ImageN]` and may contain
  `[AudioN]` / `[0-5s]` segment tokens (Seedance style) — phrasing prompts the same way
  works best.
- **Gemma truncation**: very long multi-segment prompts may be truncated by the text
  encoder's max length, dropping later segments.
